Sala de Estudo Inteligente

Sistema distribuído de baixo custo para monitorização de ocupação e conforto ambiental em salas de estudo universitárias. Baseado em ESP32, YOLOv11x e sensores IoT, com previsão de série temporal recente. Sistema-piloto instalado na Biblioteca Geral de Gualtar.

Arquitetura

Três camadas independentes que comunicam apenas pelo Firebase. Cada peça pode falhar isoladamente sem derrubar o sistema todo.

Camada de sensorização

Vision Node ESP32-CAM AI-Thinker · OV2640 Captura SVGA a cada 15 s
Sensor Node ESP32-S3 DevKitC-1 DHT11 · MQ-135 · LM393 · MSM261 I2S LED semáforo (R/Y/G)
↓ Wi-Fi 2.4 GHz

Camada de armazenamento (Firebase)

Vision Project Storage: imagens JPEG (privacy by design) RTDB: latest_image, last_capture
Sensor Project RTDB: environment, occupancy Histórico push contínuo
↓ Firebase Admin SDK

Camada de processamento (PC local)

detector.py YOLOv11x (109 MB) — pessoas + mobília Thresholds per-class · Loop 5s
forecast_service.py Holt-Winters (sazonal) → SES → Naive 1h adiante a cada chamada
api.py (Flask) REST · 5 endpoints · CORS 5/3 estados de ocupação
↓ HTTP localhost:5000

Camada de apresentação

Dashboard Web HTML/CSS/JS · Chart.js · Mock fallback
App Android Kotlin · Mesma API
LED na sala Sensor_NODE lê status a cada 5 s Verde · Amarelo · Vermelho

Sensores do nó ambiental

Sensor Medição Interface GPIO Notas
DHT11 Temperatura, humidade 1-wire digital 4 ±2 °C / ±5 %; pull-up no módulo
MQ-135 Qualidade do ar (VOCs, CO₂) ADC 12-bit (analógico) 5 Alimentado a 5 V; divisor de tensão 10k+10k para 3.3 V
LM393 (fotodíodo) Iluminância ADC analógico + digital 6, 7 ADC inverso — menor = mais luz
MSM261S4030H0 Ruído (nível RMS em dB rel.) I2S MEMS digital 24-bit 13, 14, 15 SEL→GND (canal esquerdo); 3.3 V apenas
LED Vermelho Indicador "cheio" PWM digital 16 220 Ω em série; cátodo a GND
LED Amarelo Indicador "parcial" PWM digital 17 idem
LED Verde Indicador "livre" PWM digital 18 idem

Stack tecnológico

Firmware
  • Arduino framework para ESP32 (PlatformIO)
  • Firebase ESP Client (mobizt)
  • DHT sensor library (Adafruit)
  • ESP-IDF i2s driver (microfone MEMS)
Processamento (PC)
  • Python 3.11+, PyTorch (CPU)
  • Ultralytics YOLOv11x
  • OpenCV, NumPy, Pandas
  • statsmodels (Holt-Winters, SES)
  • Flask + flask-cors (API)
  • firebase-admin (Storage + RTDB)
Frontend Web
  • HTML5, CSS3 puro
  • JS vanilla (sem framework)
  • Chart.js 4 + adaptador date-fns
  • Font Awesome 6 (ícones)
  • Tipografia Source Serif + Source Sans
Cloud
  • 2 × Firebase Project (Vision + Sensor)
  • Realtime Database (europe-west1)
  • Cloud Storage (privacy by design)
  • Authentication Email/Password

Componente preditivo

O serviço de forecasting escolhe automaticamente o modelo mais sofisticado que os dados disponíveis permitem:

  1. Holt-Winters aditivo com sazonalidade diária (1440 minutos) — requer ≥ 2 dias de histórico
  2. Simple Exponential Smoothing + tendência local — funciona com ≥ 15 minutos
  3. Naive (repete último valor) — fallback honesto quando não há dados

O notebook em processing/ml/ contém também uma comparação com LSTM (PyTorch) para análise offline. Previsão de ocupação está marcada como trabalho futuro (requer semanas de histórico real).
